# Записка до курсової роботи

## Опис файлів

### Документи

1. **записка_курсова.tex** - LaTeX документ з повним описом курсової роботи
   - Детальний опис всіх 12 шаблонів проектування
   - Приклади коду для кожного шаблону
   - Структура системи та функціональні характеристики
   - Опис результатів роботи програми

2. **записка_курсова.html** - HTML версія документа
   - Повна копія LaTeX документа в HTML форматі
   - Готовий для перегляду в браузері
   - Підтримка друку

### Діаграми Draw.io

Всі діаграми збережені у форматі Draw.io XML і можуть бути відкриті в:
- [draw.io](https://app.diagrams.net/) (онлайн)
- Draw.io Desktop версія
- Visual Studio Code з розширенням Draw.io

#### Окремі діаграми шаблонів:

1. **observer_pattern.drawio** - Діаграма шаблону Observer (Спостерігач)
   - Показує структуру ITicketObserver, TicketSubject
   - Конкретні реалізації: LoggingObserver, JsonSaveObserver

2. **visitor_pattern.drawio** - Діаграма шаблону Visitor (Відвідувач)
   - Показує ITicketVisitor та JsonSerializationVisitor
   - Відносини з Ticket, Route, Passenger

3. **strategy_pattern.drawio** - Діаграма шаблону Strategy (Стратегія)
   - Показує IPaymentStrategy та його реалізації
   - CashPaymentStrategy, CardPaymentStrategy, OnlinePaymentStrategy
   - Використання в TicketFactory

4. **factory_pattern.drawio** - Діаграма шаблону Factory (Фабрика)
   - Показує ITicketFactory та його реалізації
   - RegularTicketFactory, DiscountTicketFactory, ChildTicketFactory
   - TicketFactoryProvider для отримання фабрик

5. **builder_pattern.drawio** - Діаграма шаблону Builder (Будівник)
   - Показує TicketBuilder з ланцюжковими методами
   - Процес створення Ticket об'єкта
   - Приклад використання в Client

6. **command_pattern.drawio** - Діаграма шаблону Command (Команда)
   - Показує ICommand та його реалізації
   - BuyTicketCommand, CancelTicketCommand
   - Використання Invoker та ITicketRepository

7. **singleton_repository_pattern.drawio** - Діаграма шаблонів Singleton + Repository
   - Показує ITicketRepository та JsonTicketRepository
   - Singleton реалізація з thread-safe Instance
   - Використання TicketData для збереження

8. **decorator_pattern.drawio** - Діаграма шаблону Decorator (Декоратор)
   - Показує ITicketDecorator та його реалізації
   - InsuranceDecorator (+50 грн), BaggageDecorator (+30 грн)
   - Приклад комбінування декораторів

9. **template_method_pattern.drawio** - Діаграма шаблону Template Method
   - Показує абстрактний TicketProcessor
   - Конкретна реалізація StandardTicketProcessor
   - Шаблонний метод ProcessTicket з кроками

10. **state_pattern.drawio** - Діаграма шаблону State (Стан)
    - Показує ITicketState та три стани
    - ActiveTicketState, CancelledTicketState, UsedTicketState
    - TicketStateContext для управління переходами

11. **composite_pattern.drawio** - Діаграма шаблону Composite (Компонувальний)
    - Показує IMenuComponent та його реалізації
    - MenuItemLeaf (листок), MenuItemComposite (композит)
    - MenuBuilder для побудови меню

12. **complete_class_diagram.drawio** - Повна діаграма класів системи
    - Всі 12 шаблонів проектування
    - Всі класи моделей (Ticket, Route, Passenger)
    - Всі взаємозв'язки між класами
    - Повна архітектура системи

## Як використовувати

### Компіляція LaTeX документа

1. Встановіть LaTeX дистрибутив (TeX Live, MiKTeX)
2. Встановіть необхідні пакети (вказані в преамбулі)
3. Компілюйте командою:
   ```bash
   pdflatex записка_курсова.tex
   ```
4. Для правильної нумерації запустіть двічі

### Перегляд HTML документа

Просто відкрийте файл `записка_курсова.html` у будь-якому браузері.

### Відкриття діаграм Draw.io

1. Відкрийте [draw.io](https://app.diagrams.net/)
2. File → Open → виберіть файл `.drawio`
3. Або перетягніть файл на сторінку draw.io

### Експорт діаграм

У Draw.io можна експортувати діаграми в:
- PNG (для вставки в документ)
- PDF (для друку)
- SVG (векторний формат)
- LaTeX (для вставки в LaTeX документ)

## Структура документа

1. **Титульна сторінка** - стандартна форма курсової роботи
2. **Зміст** - автоматично генерується
3. **Перелік умовних позначень** - скорочення та терміни
4. **Вступ** - мета та завдання роботи
5. **Розділ 1** - Структурно-алгоритмічна організація
   - 1.1. Модульна організація програми
   - 1.2. Функціональні характеристики
6. **Розділ 2** - Програмна реалізація
   - 2.1. Обґрунтування вибору та опис шаблонів (детально кожен з 12)
   - 2.2. Діаграма класів
   - 2.3. Опис результатів роботи програми
7. **Висновки** - підсумки роботи
8. **Список літератури** - використані джерела

## Особливості

### Детальний опис кожного шаблону включає:

- **Призначення** - для чого використовується шаблон
- **Обґрунтування вибору** - чому саме цей шаблон
- **Структура реалізації** - код з коментарями
- **Конкретні реалізації** - класи та інтерфейси
- **Приклади використання** - як застосовується в системі
- **Переваги** - що дає використання шаблону

### Діаграми показують:

- Всі класи та інтерфейси
- Відносини наслідування (стрілки з трикутником)
- Відносини реалізації (пунктирні стрілки)
- Композицію та агрегацію (ромби)
- Залежності між класами

## Примітки

- Діаграми створені у форматі Draw.io XML
- Для експорту в PNG/PDF відкрийте діаграми в draw.io
- LaTeX документ потребує встановлених пакетів
- HTML версія готова до використання без додаткових інструментів
